# Pear вЂ” AI Assistant Guide

## What This App Is

Pear (package `hane`, slug `wingmate`) is an iOS dating app. Two roles:

- **Dater** вЂ” browses profiles, swipes, chats with matches.
- **Winger** вЂ” a trusted friend who swipes on behalf of a dater.

Core flows: Auth в†’ Onboarding в†’ Discover в†’ Matches в†’ Messaging в†’ Wingpeople.

## How to Add a New Feature

1. **SQL migration** in `supabase/migrations/<timestamp>_<feature>.sql` вЂ” always enable RLS.
2. **`npm run codegen`** вЂ” regenerates `types/database.ts`, `db/schema.ts`, `openapi.json`, `lib/api/generated/`.
3. **Domain** under `supabase/functions/api/domains/<feature>/`:
   - `schemas.ts` вЂ” Zod shapes tagged `.openapi(name)`. Derive enum literals from `pgEnum.enumValues`.
   - `queries.ts` вЂ” Drizzle fetches; first arg always `db: DBOrTx`. No Hono/Zod imports.
   - `transformers.ts` вЂ” snake_case в†’ camelCase row mappers.
   - `route.ts` вЂ” `createRoute` + `mount<Feature>(app)`. Use `getDeps(c)` for `{ db, userId, push }`.
   - Register `mount<Feature>(app)` in `app.ts`.
4. **`npm run codegen`** again to pick up the new route.
5. **Screen** вЂ” import generated hook: `useGetApi<Feature>Suspense()` from `@/lib/api/generated/<tag>/<tag>.ts`. Destructure `data` directly вЂ” no wrapper to peel.
6. **Mutations** вЂ” call the generated function directly: `await postApi<Feature>(body)` returns the success body or throws. Wire with `useMutation` + `invalidateQueries`.
7. **Commit** generated artifacts in the same commit as the source that produced them.

## Routing & Auth Gate

`RootNavigator` in `app/_layout.tsx` reads `session` from `useSession()`:

- No session в†’ `/(auth)/login`
- No `chosen_name` в†’ `/(onboarding)`
- Role = `winger` в†’ `/(winger-tabs)`
- No `datingProfile` в†’ `/(onboarding)`
- Otherwise в†’ `/(tabs)/discover`

Two `useEffect`s allowed in `AuthenticatedNavigator`: deep-link invite check from AsyncStorage, and push token registration вЂ” both genuine external events.

**Hard constraints:**

- Never run `drizzle-kit generate` or `drizzle-kit push`. Schema writes go through SQL migrations only.
- After `db:drizzle` introspects, strip the `profiles.id в†’ auth.users` FK (not in `public` schema).
- Generated artifacts belong in the same commit as the source change.

## Edge Functions Architecture

`supabase/functions/api/` is a single Hono app. Key points:

- **Auth middleware** sets `c.var.{userId, token, claims}` from JWT. Kong verifies in prod; skipped locally with `--no-verify-jwt`.
- **Transaction middleware** opens a Drizzle tx per request, sets `role = authenticated` + `request.jwt.claims` for RLS, exposes as `c.var.db`. Commits on `return`, rolls back on `throw`. A returned 4xx **commits**; a thrown `HTTPException` rolls back.
- **Never import the module-level `db`** from `db/client.ts` in handlers вЂ” always use `c.var.db` via `getDeps(c)`.
- **Push** via `c.var.push.send(token, title, body)`. Storage via `c.var.supabase`. No new standalone edge functions вЂ” side effects belong in Hono handlers.
- **RLS is the auth floor.** Handler-side `where(eq(...))` clauses are for correctness/perf, not security.
- **Secrets:** `DATABASE_URL` (authenticator role, Supavisor in prod, `host.docker.internal:54322` locally). `SUPABASE_URL/ANON_KEY/REGION` auto-injected. `SUPABASE_REGION` absence = local dev (push no-ops).

## Styling (NativeWind v5 + Tailwind v4)

- **Always** import styled primitives from `@/lib/tw`: `View`, `Text`, `Pressable`, `ScrollView`, `TextInput`, `SafeAreaView`, `AnimatedView`.
- Never use `StyleSheet.create` вЂ” use `className`.
- `style` prop only for: `borderCurve`, dynamic/animated values, SVG props.
- Never use function-style `style` on `Pressable` (NativeWind bug #1105 вЂ” children don't render). Use `active:` pseudo-class.
- `cn()` from `@/lib/cn` for conditional classes.
- Design tokens in `global.css` under `@theme` вЂ” never add raw hex to `constants/theme.ts`.

**Modal color caveat:** CSS variables aren't injected in Modal's native layer. Use `ModalView` from `@/lib/tw` with a `backgroundColor` style prop (not className) for colors inside Modals.

## Coding Preferences

- **Composition:** Orchestrators decide what to render; logic lives close to where it's used.
- **Control flow:** `switch` on discriminated unions. Explicit `return` on every branch.
- **No `useEffect`:** Move async work into transition handlers. Exceptions: Supabase auth subscription, push token registration, AsyncStorage deep-link check (mount-only, genuine external events).
- **Error propagation:** No try/catch across callback boundaries. Return errors as values. User-facing errors via `toast.error()`.
- **Forms:** react-hook-form everywhere вЂ” `Controller`, `handleSubmit`, `isSubmitting`, `isValid`, `mode: 'onChange'`.
- **Queries:** Transforms belong in the query function. Unwrapping boilerplate belongs in the query wrapper, not callsites. No magic string cache keys.
- `async/await` over `.then()`.

## Development Workflows

The repo is a monorepo. All app commands run from `app/` вЂ” either via the root `Justfile`
(`just --list` to see recipes) or directly with `cd app && npm run вЂ¦`. Migration plan and the
backend/infra rollout live in `docs/migration/` (read it before backend work).

```bash
# App вЂ” via Justfile (from repo root)
just app-dev          # iOS Simulator (cd app && npm run dev:sim)
just app-web          # Expo web (auto-signs in as dev@local.test)
just app-lint         # cd app && npm run lint
just app-typecheck    # cd app && npx tsc --noEmit
just app-codegen      # cd app && npm run codegen (regenerate all artifacts)

# App вЂ” directly (cwd must be app/)
cd app && npm run dev:sim
cd app && npm run web

# Supabase (legacy backend, still live вЂ” scripts cd to repo root internally)
cd app && npm run db:up / db:down / db:reset / db:fixtures
cd app && npm run api:serve   # hot-reload api edge function (--no-verify-jwt locally)
cd app && npm run codegen     # regenerate all artifacts

# Verify (run before committing вЂ” from app/)
cd app && npm run lint && npm run typecheck:functions && npm run lint:functions && npx tsc --noEmit

# Build & deploy (from app/)
cd app && npm run build:local / build:device
cd app && npm run deploy:ota     # EAS OTA update
cd app && npm run deploy:build   # EAS production build
```

`app/.env.local` needs: `EXPO_PUBLIC_SUPABASE_URL`, `EXPO_PUBLIC_SUPABASE_ANON_KEY`, `DATABASE_URL=postgresql://authenticator:postgres@host.docker.internal:54322/postgres`, `SUPABASE_ANON_KEY`.
